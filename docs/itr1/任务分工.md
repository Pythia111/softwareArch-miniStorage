### 成员2：目录管理 + LS命令

cx

- **负责文件**：`Directory.java`、`LsCommand.java`
- **具体任务**：
  1. 实现`Directory`类：使用`TreeMap<String, Node>`存储子节点（自动按字母序排序）。
  2. 提供`putChild(String name, Node node)`、`getChild(String name)`、`removeChild(String name)`、`listChildren()`（返回已排序的名称列表）。
  3. 实现`Directory`的`getSize(SizeContext ctx)`：遍历所有子节点，累加每个子节点的`getSize(ctx)`（注意调用时传递同一个`ctx`实例以支持未来防环）。
  4. 实现`LS`命令的逻辑：根据路径获取节点，如果是目录则输出`listChildren()`的每一行，如果是文件则输出文件名。
  5. 确保`LS`对空目录输出空（无输出），对根目录正常工作。

**工作量评估**：`TreeMap`管理 + 递归遍历 + 命令实现，与成员1相当。

### 成员3：路径工具 + MKDIR命令

cyg

- **负责文件**：`PathUtil.java`、`MkdirCommand.java`
- **具体任务**：
  1. 实现`PathUtil.normalize(String absPath)`：去除多余斜杠（如`///a//b` → `/a/b`），迭代一先处理简单情况，预留`. / ..`扩展点（注释说明）。
  2. 实现`split(String absPath)`：返回路径组件数组（如`/a/b/c` → `["a","b","c"]`）。
  3. 实现`getParentPath(String absPath)`和`getBaseName(String absPath)`。
  4. 实现`MKDIR`命令的逻辑：解析绝对路径，得到父路径和待创建目录名。检查父目录是否存在且为目录，若存在则创建新的`Directory`实例并加入父目录；否则静默忽略（无输出）。需要调用成员2提供的`Directory`类的`putChild`方法。
  5. 处理根目录：若路径本身就是`/`，忽略（不能创建根目录）。

**工作量评估**：路径处理算法（中等难度）+ 命令实现（需要与其他成员协作），整体工作量适中。

## 协作接口约定（确保无缝对接）

| 公共组件 | 提供者 | 关键方法签名 |
|---------|--------|-------------|
| `Node` | 成员1 | `abstract long getSize(SizeContext ctx)` |
| `File` | 成员1 | `File(String name, int size)` |
| `SizeContext` | 成员1 | `void addVisited(Node n)`, `boolean isVisited(Node n)` |
| `Directory` | 成员2 | `void putChild(String name, Node node)`, `Node getChild(String name)`, `Collection<String> listChildren()` |
| `PathUtil` | 成员3 | `static String[] split(String absPath)`, `static String getParentPath(String absPath)` |
| `MemFs` | 成员4 | `void mkdir(String path)`, `void touch(String path, int size)`, `List<String> ls(String path)`, `long info(String path)` |

- 所有路径均为绝对路径，成员3的`PathUtil`负责解析。
- 成员1、2、3需确保自己的类不依赖`Main`或`MemFs`，只暴露必要的公共方法。
- 成员4在集成时可能需要微调调用方式，应提前一周进行联调。
